Elementos Clave de Arquitectura Móvil

Compendio técnico de terminología fundamental para el diseño y desarrollo de aplicaciones modernas.

Presentación
UI (User Interface)
Punto de interacción entre el usuario y el sistema. En arquitectura, debe ser "tonta" (passive), limitándose a renderizar datos y capturar eventos sin procesar lógica de negocio.
Presentación
View State
Representación de datos inmutables que describen exactamente cómo debe verse la pantalla en un momento dado (ej: cargando, lista de usuarios, error).
Lógica
Business Logic
Conjunto de reglas y algoritmos que definen el funcionamiento del núcleo de la aplicación, independiente de cómo se muestren los datos o dónde se guarden.
Lógica
ViewModel / Presenter
Intermediario que transforma los datos crudos del modelo en información lista para ser mostrada por la vista, gestionando el estado y la lógica de presentación.
Datos
Repository Pattern
Patrón que abstrae el origen de los datos. La app pide datos al repositorio sin saber si vienen de una base de datos local (cache) o de una API remota (nube).
Datos
Single Source of Truth
Principio que establece que debe haber una única fuente confiable para cada dato en la app, evitando conflictos de sincronización entre diferentes pantallas.
Flujo
Unidirectional Data Flow
Patrón donde los datos fluyen en una sola dirección (Estado -> Vista) y los eventos en otra (Vista -> Intención -> Estado), haciendo el sistema predecible y fácil de depurar.
Flujo
Dependency Injection
Técnica donde los objetos reciben sus dependencias desde fuera en lugar de crearlas ellos mismos, facilitando el testing y el desacoplamiento de componentes.
Ciclo de Vida
Lifecycle Awareness
Capacidad de los componentes de arquitectura para saber en qué estado está la pantalla (activa, pausada, destruida) y actuar en consecuencia para evitar fugas de memoria.
